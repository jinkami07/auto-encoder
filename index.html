Section 01

Autoencoder (AE) とは何か

入力を低次元の潜在空間に圧縮し、そこから元の入力を再構成することを学ぶニューラルネットワーク。 目的は「圧縮しても失われない特徴表現を得ること」。

AE アーキテクチャ
入力
x ∈ ℝⁿ
→
Encoder
f_φ(x)
→
潜在変数
z ∈ ℝᵈ (d ≪ n)
→
Decoder
g_θ(z)
→
再構成
x̂ ∈ ℝⁿ

損失関数

# 画像など連続値 → MSE L = ||x - x̂||² = ||x - g_θ(f_φ(x))||² # バイナリデータ → Binary Cross Entropy L = -Σ [ x log(x̂) + (1-x) log(1-x̂) ]
AE が学ぶこと

各入力を再構成しやすいような潜在ベクトル z を学習します。点推定のため、潜在空間に構造的な制約はありません。

AE の弱点

潜在空間が「穴だらけ」になりやすく、ランダムに z をサンプリングしても意味のある出力は得られません。生成モデルとしては不向きです。

💡
ボトルネック構造がポイント： 潜在次元 d を入力次元 n より小さく絞ることで、ネットワークは「本質的な特徴」だけを残す表現を学ばざるを得なくなります。
Section 02

Variational Autoencoder (VAE) とは何か

潜在変数を「点」ではなく「確率分布」として扱うことで、滑らかに構造化された潜在空間を獲得するモデル。 生成モデルとしても機能します。

VAE アーキテクチャ
入力
x
→
Encoder
μ(x), logσ²(x)
→
Sample
z = μ + σε
→
Decoder
p(x|z)
→
再構成
x̂

損失関数 — ELBO の最大化

# ELBO (Evidence Lower Bound) を最大化 ＝ 負の ELBO を最小化 L = -ELBO = L_recon + L_KL = -E[log p_θ(x|z)] + D_KL(q_φ(z|x) || N(0, I)) # KL 項の解析解（正規分布同士） L_KL = -0.5 * Σ ( 1 + logσ² - μ² - σ² )
📐
KL Divergence の役割： 各データ点の潜在分布 q(z|x) を標準正規分布 N(0, I) に近づける罰則です。 これにより潜在空間が整えられ、ランダムサンプリングが可能になります。
VAE が学ぶこと

データの背後にある確率的な潜在構造を学習します。潜在空間は連続的かつ滑らかになり、補間・生成が可能です。

VAE のトレードオフ

KL 項が強すぎると再構成が曖昧になります（β-VAE の β パラメータで調整）。AE より再構成がぼやける傾向があります。

Section 03

Reparameterization Trick

確率的サンプリングを挟みながら逆伝播を可能にする、VAE の核心テクニック。

なぜ必要か

ナイーブに z ~ q(z|x) とサンプリングすると、その操作は微分不可能になり、 Encoder の重みに勾配が流れません。

⚠️
問題： サンプリング操作は決定論的でないため、そのままでは ∂z/∂φ が計算できません。

解決策

# ε を「外から入れる」乱数として分離 ε ~ N(0, I) # 微分不要な乱数 z = μ + σ ⊙ ε # μ と σ には勾配が流れる

乱数 ε を外から注入することで、z の計算グラフを μ, σ に対して微分可能な形に変換します。

Before vs After
❌ ナイーブな方法（逆伝播不可）
z = sample(μ(x), σ(x))
← 勾配が μ, σ に届かない
✅ Reparameterization（逆伝播可能）
ε ~ N(0, I)
z = μ(x) + σ(x) ⊙ ε
← μ と σ に勾配が流れる
✅
ポイント： 確率性（ランダム性）を ε に押し込み、学習可能なパラメータ μ, σ だけに勾配を流すのがこのトリックの本質です。
Section 04

AE と VAE の比較

構造・損失・潜在空間・生成能力の観点から両者の違いを整理します。

観点 Autoencoder (AE) Variational Autoencoder (VAE)
Encoder の出力 潜在ベクトル z を直接出力 分布パラメータ μ, logσ² を出力
潜在変数の扱い 決定論的な点推定 点 確率分布からのサンプリング 分布
損失関数 再構成誤差のみ
||x - x̂||²		 再構成誤差 + KL Divergence
L_recon + D_KL(q || N(0,I))
潜在空間の構造 制約なし・不連続になりやすい 標準正規分布に近い連続的な空間
生成能力 弱い — ランダム z は意味不明な出力 強い — N(0,I) からサンプリングで生成可
再構成の鋭さ シャープ（点推定のため） やや曖昧（確率的サンプリングのため）

Section 05

どちらを選ぶべきか

タスクの目的によって AE と VAE のどちらが適切かが変わります。

AE を選ぶ場面
  • 次元削減・圧縮 — 高次元データをコンパクトに表現したい
  • 異常検知 — 正常データの再構成誤差を基準にする
  • ノイズ除去 — 入力にノイズ、出力にクリーンな画像（Denoising AE）
  • 特徴抽出の前処理 — 下流タスク用の特徴表現として
  • シンプルさ優先 — 実装・チューニングが容易
異常検知 圧縮 特徴抽出 Denoising
VAE を選ぶ場面
  • データ生成 — 既存データに似た新しいサンプルを作りたい
  • 潜在空間の補間 — クラス間を連続的に変化させたい
  • Disentanglement — 潜在変数の各次元に意味を持たせたい
  • 不確実性の定量化 — 確率的な予測が必要
  • 半教師あり学習 — ラベルなしデータを活用したい
生成 補間 Disentangle 半教師あり
🎯
一言まとめ： 「忠実に圧縮・再構成したい」なら AE。「新しいデータを生成したい・潜在空間を探索したい」なら VAE。
Section 06

実装で何が変わるか

AE を VAE に変えるために必要な変更は 4 ステップだけです。

1

Encoder の出力を 2 本に分ける

z を直接返す代わりに mu と logvar を返します。

2

Reparameterization を実装する

z = mu + eps * std の形でサンプリングします。

3

損失関数に KL 項を追加する

再構成誤差だけでなく KL Divergence を合計します。

4

評価方法を変える

再構成品質だけでなく、ランダムサンプリングの品質も見ます。

AE → VAE の変更点（PyTorch）

## AE の Encoder def forward(self, x): z = self.encoder(x) # 点推定 x_hat = self.decoder(z) return x_hat ## VAE の Encoder（変更点） def encode(self, x): h = self.encoder(x) mu = self.fc_mu(h) # ① μ を出力 logvar = self.fc_logvar(h) # ① logσ² を出力 return mu, logvar def reparameterize(self, mu, logvar): std = torch.exp(0.5 * logvar) eps = torch.randn_like(std) # ② ε をサンプリング return mu + eps * std # ② z = μ + σε ## VAE の損失（変更点） recon_loss = F.mse_loss(x_hat, x) kl_loss = -0.5 * torch.mean( # ③ KL 項 1 + logvar - mu**2 - logvar.exp() ) loss = recon_loss + kl_loss

Section 07

まとめ

🗜️
AE = 圧縮器

うまく圧縮して再構成するモデル。潜在空間は点の集合で、構造的な保証はない。

🎲
VAE = 確率的生成モデル

潜在空間を確率分布として整え、ランダムサンプリングで新しいデータを生成できる。

🔑
鍵は「点 vs 分布」

AE は潜在変数を「点」として扱い、VAE は「分布」として扱う。この違いがすべての違いの源泉。

📌
一行まとめ： AE は「再構成重視の表現学習」、VAE は「生成と滑らかな潜在空間のための確率的モデリング」。 両者の本質的な違いは 潜在変数を確率分布として扱うかどうか にあります。
