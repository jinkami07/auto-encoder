Autoencoder と VAE の違いを、教科書的に整理する

「入力を圧縮して、また元に戻す」という見た目は似ていますが、 Autoencoder と Variational Autoencoder は学んでいるものがかなり違います。 このページは、実装より先に考え方を腹落ちさせるための整理です。

1. まず Autoencoder とは何か

Autoencoder は、入力 x をいったん低次元の潜在表現 z に圧縮し、 そこから入力を再構成するモデルです。目的は「元のデータをなるべく失わずに表現すること」です。

入力画像
x
→
Encoder
z = f(x)
→
Decoder
x̂ = g(z)
Loss = Reconstruction Error = ||x - x̂||² あるいは BCE(x, x̂)

何を学ぶか

各入力を再構成しやすいような潜在ベクトル z を学びます。

弱点

潜在空間がばらばらでも困らないため、ランダムに z を振ると意味のある画像が出る保証はありません。

2. VAE は何を追加したのか

VAE は「潜在空間をきれいな確率分布にそろえる」という制約を加えたモデルです。 そのため、Encoder は 1 本のベクトルではなく、平均 μ と分散の情報 logσ² を出します。

Loss = Reconstruction Loss + KL Divergence

KL 項は、「各データごとの潜在分布 q(z|x) が標準正規分布 N(0, I) に近づくようにする罰則」です。

3. Reparameterization Trick の意味

VAE では z を確率的にサンプリングしたいのですが、単純に乱数を挟むと逆伝播がしづらくなります。 そこで次の形に書き換えます。

ε ~ N(0, I)
z = μ + σ ⊙ ε

こうすると、乱数は ε に押し込みつつ、μ と σ に対して勾配を流せます。 これが reparameterization trick です。

4. AE と VAE の違いを表で整理

観点 Autoencoder VAE
Encoder の出力 z を直接出す μ と logσ² を出す
潜在空間 自由に散らばりやすい 標準正規分布に近づくよう整えられる
損失 再構成誤差のみ 再構成誤差 + KL divergence
生成のしやすさ 弱い 強い
再構成の鋭さ 比較的シャープになりやすい ややぼやけることがある
向いている用途 圧縮、特徴抽出、異常検知の前処理 生成、潜在空間の補間、分布としての学習

5. あなたのノートブックが VAE でなかった理由

元のノートブックの特徴

  • encoder(x) がそのまま潜在変数 z を返す
  • 損失が MSELoss だけ
  • KL 項がない
  • mu, logvar, reparameterize がない

つまり何か

それは「通常の Autoencoder」であり、VAE ではありません。

6. 実装で何を変えれば VAE になるか

1

Encoder の最後を 2 本に分ける

z を 1 本返す代わりに、mu と logvar を返します。

2

サンプリングを書く

z = mu + eps * std の形で reparameterization を入れます。

3

損失に KL 項を足す

再構成誤差だけでなく、潜在分布を整える罰則を足します。

4

評価の見方を変える

再構成の良さだけでなく、潜在空間の滑らかさとサンプリング結果も見ます。

7. 学ぶときの見方

AE を見るポイント

  • 再構成がどこまで忠実か
  • 潜在空間でクラスが分かれているか
  • 次元圧縮として使えるか

VAE を見るポイント

  • 潜在空間が滑らかにつながるか
  • ランダムサンプリングで自然な画像が出るか
  • KL が強すぎて再構成が崩れていないか

8. 結論

Autoencoder は「うまく圧縮して戻すモデル」、VAE は「確率分布として潜在空間を整えながら生成もできるモデル」です。 両者の一番大きな違いは、VAE が潜在変数を点ではなく分布として扱うことにあります。

そのため、AE は再構成重視、VAE は生成と滑らかな潜在空間重視、と覚えると整理しやすいです。
